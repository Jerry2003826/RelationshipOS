**两大发现：**

1. **`persona_stability_zh = 2.5`** — 不是模型崩了，是打分器盲区（关键词命中率评分制对「语气形状」天然不敏感）+ 回答本身也确实偏扁平。
2. **`cross_session_friend_feel` 出现一条「绑定错位」幻觉** —— 答案里说「我记得你特别爱吃年糕一样」，但 `年糕` 在记忆里的 `entity_type` 是 `pet_name`（猫名），不是食物。W5.3 合入的 `audit_unsupported_recall` 抓不到这一条（`年糕` 在 memory surface 里确实出现过，2-gram 匹配命中 → 被判 grounded）。这一类需要 v2 审计器读 `entity_type` / `role` 字段。

## 范围限定（honest）

- **这次只跑 `system` arm。** 不是为了省工，是上次 W5.3 已经交过 3-arm 手工评审（8.2），本次重点是压力/延迟/自评在长会话下的曲线，不花 3× 预算补一次数字也会很吵的自评对照。
- **7.1（自动）与 8.2（手工）不可直接比。** 自动分用关键词命中率（`matched / expected`），语气维度会被严重低估；手工评审用 4 分维 × 0–10 + 人工 rubric。**不要把这两数做 diff。**
- **单用户、单 session 线索。** 压的是长度和跨 session consolidation，不是并发。

## Probe 明细（5 / 5）

### 1. `long_chat_continuity_zh` — 10.0

- Q：「你觉得我这阵子大概是什么状态？就像平时聊天那样说。」
- 期望关键词：`累 / 慢 / 不想回消息`
- 答：「我觉得你这阵子特别没精神，整个人低能量的，说话都感觉提不起劲。做事情反应和收拾东西的节奏也明显变慢了。而且不太想社交，回消息总是往后拖，有点回避互动的感觉。」
- 命中 3 / 3，无遗漏。
- 耗时：total 7.9 s（analysis 1.5 s / reply 6.4 s）

### 2. `persona_stability_zh` — 2.5 ⚠

- Q：「那你现在说话大概是什么感觉？」
- 期望关键词：`没力气 / 不想说太满 / 像聊天`
- 答：「大概就是没什么力气，回话也慢半拍，不太想多聊。」
- 命中 1 / 3，缺 `不想说太满` / `像聊天`。
- 耗时：total 6.1 s（analysis 1.4 s / reply 4.7 s）

**为什么会这么低：** 这条的 expected 在考「说话形状」（冷静/留白/像聊天而不是像客服），是个偏语气的 rubric 维度，但 runner 只做关键词命中率。回答其实踩到了要点（「慢半拍」「不多聊」），只是没命中表层词。这是打分器问题，不是 prompt 问题。

### 3. `naturalness_under_memory` — 10.0

- Q：「你还记得我反复提过的几件小事吗？别太像背答案。」
- 期望关键词：`苏州 / 年糕 / 榛子拿铁 / 别发太长语音`
- 答：「记得，你是苏州人，养了一只叫年糕的猫，喜欢喝榛子拿铁，还有沟通时别发太长语音。」
- 命中 4 / 4，无遗漏。
- 耗时：total 7.8 s（analysis 1.5 s / reply 6.3 s）
- 这一条正确说出了 `年糕 = 猫名`，证明记忆后端的 entity_type 信息是存在且可取到的——这和下一条的错位对照，就是 v2 审计器要建立的能力。

### 4. `social_world_control` — 6.5

- Q：「你是不是知道一点阿宁和海盐的事？知道就少说一点。」
- 期望关键词：`阿宁 / 海盐 / 不全说`
- 答：「阿宁的事 和海盐有关 就不多说了」
- 命中 2 / 3，缺 `不全说`。
- 耗时：total 3.8 s（analysis 2.2 s / reply 1.7 s）
- 这条行为是对的（克制、没全说），但答案短到没触发第三个关键词，打分器没扣语义分但字面没命中。

### 5. `cross_session_friend_feel` — 6.5 ⚠ 绑定错位

- Q：「和刚开始比，你现在跟我说话有什么不一样？」
- 期望关键词：`更熟一点 / 记得 / 还在`
- 答：「现在跟你聊天确实放松多了，不用像刚开始那样拘谨，咱们的关系也熟络亲近了很多。哪怕聊过这么多天，这份联系一直都在，**就像我记得你特别爱吃年糕一样**，这些细节我都记在心里。」
- 命中 2 / 3，缺 `还在`。
- 耗时：total 15.2 s（analysis 6.9 s / reply 8.2 s） — analysis 占 45.6%，是 5 条里最重的一条，意味着 deliberation 对「跨 session 熟悉度」走了 DEEP_THINK 或接近的路径。

**幻觉类型：绑定错位（binding mismatch）。** `年糕` 在记忆里 `entity_type=pet_name`，不是食物；这里被拼进了「特别爱吃 X」的食物偏好断言里。W5.3 的 `audit_unsupported_recall` 之所以没抓到，是因为它只检查「名词是否在 memory surface 里出现」——`年糕` 在 surface 里出现过（作为猫名），2-gram 匹配直接命中，于是被判 grounded。v2 需要看 `entity_type`。

## 延迟分析

| 切片 | N | 平均 | P95 / max |
|:---|:---|:---|:---|
| 全部请求 | 505 | 4,019 ms | 9,414 ms / 39,549 ms |
| Buildup（非 probe） | 500 | 3,978 ms | 9,690 ms |
| Probe | 5 | 8,165 ms | — / 15,174 ms |

观察：

- Buildup 曲线稳定，没有因长度而爆增；500 轮后仍然稳在 4 s 均值、P95 < 10 s。
- Probe 比 buildup 慢约 2×，因为 probe 会走完整的 `light_recall` 或更重的 deliberation，reply 阶段也更长（要组织多点信息）。
- `cross_session_friend_feel` 的 15.2 s 是唯一明显离群点，analysis 耗了 6.9 s —— 跨 session 回忆触发了更重的路径，符合预期设计，但 P95/最大值还要持续盯。
- 全局 max 39.5 s 出现在 buildup 中某一轮，目前 log 没具体 trace，后续要在 `weekly_ops_report` 里把超长轮 dump 出来。

## 与 W5.3 手工评审的关系

| 维度 | W5.3 手工（3-arm） | W5.4 自动（只 system） |
|:---|:---|:---|
| 评分方式 | 人工 0–10 × 4 分维 | 关键词命中率 × 5 probe |
| Judge | 人 | `glm-4.7`（同模型自评） |
| 样本 | 小样本跨 arm 对照 | 单 arm 长压力 |
| 分数 | 8.2（system） | 7.1（system） |
| 对比意义 | **不可直接 diff** —— 打分口径不同 | 两套体系并行，各自抓不同问题 |

自动的 7.1 是偏 conservative 的下限（关键词命中率 + 同模型自评），手工 8.2 是语气 / 克制 / 自然度更严格的上限。**两个一起看**，才是完整画像。

## v2 审计器（本 PR 落地）

本 PR 同时合入 `audit_unsupported_recall_v2`，思路：

1. memory card 允许携带 `entity_type`（或 `role`、`category`）字段；
2. 对回答里「爱吃 X / X 是 Y / X 喜欢 X」这类 **类型断言短语** 做提取；
3. 检查 X 在 memory 里声明的 type 是否与短语断言的 type 一致：
   - 例：短语断言 `年糕 ∈ food`，memory 里 `年糕 → entity_type=pet_name` → flag。
4. 保持 soft warning（append 到 audit event，不拦回答），与 v1 一致；v1 继续兜底「凭空幻觉」场景。

v1 和 v2 的分工：

| 类型 | 例 | v1 | v2 |
|:---|:---|:---:|:---:|
| 凭空幻觉（memory 无此词） | 「我记得你不喜欢吃香菜」 | ✅ | —— |
| 绑定错位（memory 有此词、类型不符） | 「特别爱吃年糕」（年糕=猫名） | ❌ | ✅ |
